Learn to Publish your First NPM Package

Introduction

Creating a npm package might seem an incredibly scary or complicated endeavor at first (been there, done that), but once you’ve done it once you might think “Is that all? ðŸ˜•”.

Personally, I don’t like being trapped in tutorial hell, and neither should you. So, I opted to create an npm package I wanted and is useful to me outside the scope of publishing to npm. Let’s create our own npx username card, which executes a cURLable business card, because "If it's not cURLable, it's not on the web." - Vint Cerf.

Initializing the Package

I chose to name the folder and repository to be the username I use on Github, because then the README.md of the package (aka you) also acts as the profile on Github.

Initialize the Package with the NPM CLI

The npm init commands creates a package.json file, and configures the most important keys for us. Verify that the value of name does not exist in the package registry as the name is used together with the version to create a unique identifier.

Checkout the full documentation if you want to understand what the other fields are used for!

Create a index.js file

Don’t do anything fancy yet, just to see if the basic components are working together. Add something simple to index.js and place it in the root of the repository.

Verify if node can run the file you’ve created.

Publishing the Business Card to NPM

Login to NPM

In order to login to npm you first have to create an account. Do not forget to verify your email address when you created your account. It’s required before you are able to publish to npm.

Troubleshooting

If you rushed ahead (I do this myself as well ðŸ˜…) and didn’t make sure the package is available before creating your repository the publishing will fail.

Change the name of your package to a scoped one @your-user-name/business-card and make sure the scoped package’s access is set to public before running npm publish again.

Conclusion

Hurray! Now you’ve seen that the most complicated part of publishing a package is the package itself, not the npm publishing process. Additionally, people are now able to query for your business card with cURL or npx your-user-name.
